# Public / Private Boundary

## Public-safe

The following can be public after Faith review:

- project name and high-level description
- public mission and audience
- ownership and rights posture
- commercial-use policy
- roadmap summary
- public WordPress page copy
- image briefs and generated public-safe visuals
- non-sensitive status summaries

## Private by default

The following must stay private:

- source code and plugin engine
- deployment scripts and production configuration
- server paths, live architecture details, and backup paths
- credentials, secrets, tokens, API keys, OAuth secrets, and MFA material
- private prompts, agent instructions, queues, receipts, logs, and datasets
- customer, applicant, partner, family, legal, medical, benefits, housing, or
  private creative records
- unpublished manuscripts or private evidence files
- payment setup, tax details, legal filing account details, or account-console
  screenshots

## Public language guardrails

Say:

- reviewed network
- protected owner records
- owner-controlled handoffs
- preparation support
- public proof-of-concept surface

Avoid saying:

- automatic legal filing
- guaranteed protection
- investment guarantee
- official legal service
- live payment infrastructure
- public data room
- open-source release

## Release rule

Only publish contents from `_github_public_export/` after a privacy scan and Faith
review.
