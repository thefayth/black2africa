# Security

## Reporting

If you find a security issue in the public materials, contact the project owner
through an authorized private channel. Do not open a public issue containing
exploit details, secrets, private routes, customer data, server paths, or
account screenshots.

## What not to send

Do not send:

- passwords, tokens, API keys, OAuth secrets, or MFA codes
- private account screenshots
- customer or applicant records
- legal, medical, benefits, family, housing, or private creative records
- exploit scripts that target live systems

## Public-surface scope

This repository is a public documentation surface. It should not contain live
credentials, production configs, deploy scripts, source engine files, logs,
receipts, private prompts, customer data, or private records.

## Responsible handling

Security reports should include:

- file path or public URL
- short description
- impact
- reproduction steps that do not expose private information
- suggested fix, if known

The project owner reserves the right to remove unsafe disclosures.
