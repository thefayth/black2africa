# Roadmap

## Phase 1: Protected public surface

- Publish project overview.
- Publish public/private boundary.
- Publish commercial-use and rights posture.
- Prepare WordPress page draft.
- Prepare image briefs.

## Phase 2: Faith review

- Confirm public language.
- Confirm what can be described publicly.
- Remove or soften anything that feels too operational.
- Approve final repo name and visibility.

## Phase 3: Public repo launch

- Create `thefayth/black2africa`.
- Push only `_github_public_export/` contents.
- Confirm no private files are included.
- Add a repository description and topics.

## Phase 4: WordPress page

- Add draft page at `/projects/black2africa/`.
- Add featured image when approved.
- Link to the public repo after final review.
- Keep source, accounts, and private workflow links out of the page.

## Phase 5: Visual and sponsor materials

- Generate public-safe hero and social images.
- Create a short public deck if needed.
- Prepare sponsor and collaborator one-pagers without exposing private terms or
  internal records.

## Phase 6: Private engine stays private

- Keep code, prompts, logs, receipts, credentials, and private records in the
  private project system.
- Release only approved public summaries.
